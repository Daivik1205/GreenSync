## What This Project Is

**GreenSync** — an Edge AI V2X (Vehicle-to-Infrastructure) framework for urban traffic
optimization, simulated using SUMO + TraCI. Backend runs on Raspberry Pi.
Flutter app is the primary user-facing deliverable.

**Core loop:**
SUMO simulation → RSU zone sensing → MQTT → Digital Twin → AI prediction
→ Eco-routing → route fed back into SUMO (closed loop)

**IDP team:** CS + EIE + Chemical Engineering members.
ChemE member uses SQL queries on Supabase for emissions analysis.

## Key Design Decisions

- **SUMO over synthetic data**: Real map (OSM Bengaluru bbox 12.96–12.97, 77.58–77.60).
  TraCI gives realistic queue lengths, signal states, vehicle speeds.
- **XGBoost for signal phase prediction**: Tabular features, fast inference, RPi-friendly.
- **GRU for speed time-series**: Sequence model, predicts zone speed N steps ahead.
- **A\* routing**: Cost = weighted(travel_time, congestion, CO₂). Uses Digital Twin graph.
- **Closed loop**: Routing decisions fed back to SUMO via `traci.vehicle.setRoute()`.
  This is the star feature — a true simulation feedback loop.
- **Supabase over Firebase**: PostgreSQL enables relational emissions queries for ChemE team.
- **MQTT over REST**: Push-based, low latency. Fits real-time advisory use case.

## Honest Scope Warnings

- **"6G"**: Not simulated. MQTT ≠ 6G. Frame as "V2X-inspired MQTT communication".
- **"ISAC"**: No hardware. Frame as "inspired by ISAC sensing architecture".
- **"RADAR"**: Simulated via TraCI zone aggregation. Be explicit in viva.
- **"Digital Twin"**: Software state graph, not physics-based. Scope this correctly.
- **LSH**: Removed — unusual fit. If needed, use KNN instead.
- **Event propagation (Phase 7)**: Keep simple — BFS only. Don't over-engineer.
